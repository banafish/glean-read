# 提案：重构极速摘录组件 (Fast Capture Widget) UI

## 为什么

目前的 `FastCaptureActivity` 采用了基础的 Material 3 规范，虽然功能完备，但在视觉上显得过于传统且「工具感」重。用户在分享文本进行摘录时，由于弹窗样式单调且与背景契合度低，会产生明显的中断感。

为了提升 GleanRead 的用户品牌忠诚度，尤其是对高质量知识采集有追求的用户群体，我们需要将其重构为一个具有高级感、符合现代审美（参考 Notion, Obsidian, Readwise）的「心流前哨站」。

## 变更内容

- **沉浸式视觉**：实现**毛玻璃 (Glassmorphism)** 效果的底部弹窗，使界面与源应用自然交融。
- **内容为本**：引入**书摘卡片视图**，使用优雅的排版和阴影突出展示原文内容。
- **极简操作**：弱化表单边界，使用无边框输入框和胶囊标签，降低用户的采集心理负担。
- **精美反馈**：添加点击感、保存动效等细腻的微交互。

## 影响
- **代码库**：`FastCaptureActivity.kt` 的布局与逻辑重构。
- **样式系统**：可能需要新增毛玻璃、阴影、渐变等相关的 Compose 扩展或主题定义。
- **依赖**：无新增第三方库，优先使用 Jetpack Compose 原生能力实现。
