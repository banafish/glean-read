# 增量规范：fast-capture-widget（URL 感知升级）

## 变更原因

原规范中「文本分享与极速采集」场景描述"系统自动截取正文与来源 URL"，但实际实现中
URL 提取逻辑过于简单，在「选中文字分享」（主要场景）下无法提取到 URL。
本增量规范补充和明确 URL 提取相关的场景行为要求。

## 新增需求

### 需求：Intent 解析委托给 UrlExtractor

`FastCaptureActivity` 的 Intent 解析逻辑必须委托给 `UrlExtractor`，
不再包含任何内联的 URL 提取判断代码。

#### 场景：URL 提取委托

- **当** `FastCaptureActivity.onCreate` 接收到 `ACTION_SEND` Intent 时
- **那么** URL 的提取必须通过调用 `UrlExtractor.extract(intent)` 完成
- **那么** 原 L56-58 的前缀匹配逻辑**必须被移除**

### 需求：双场景 URL 感知

#### 场景：整页分享时的 URL 自动填充

- **当** 用户使用 Chrome、Firefox 等浏览器的「分享整页」功能分享到 GleanRead 时
- **那么** 弹窗必须自动填充来源 URL，展示 `SourceBadge` 域名徽章
- **那么** 不要求用户手动填写 URL

#### 场景：选中文字分享时的 URL 提取尝试

- **当** 用户选中文章中的文字通过系统「分享」发送到 GleanRead 时
- **且** 所分享的 App（如 Chrome）在文本末尾追加了原文 URL 时
- **那么** 弹窗必须识别并提取该 URL，展示 `SourceBadge`

#### 场景：URL 完全缺失时降级

- **当** 用户通过「选中文字分享」且 Intent 中不含任何可提取的 URL 时
- **那么** 弹窗必须展示折叠的 `CollapsibleUrlInput`，允许用户可选地手动填写原文链接
- **那么** 不填写 URL 也能正常保存摘录，不得阻塞主流程

## 修改需求

### 需求：系统接管与极速浮层分享（原需求补充）

对原场景「文本分享与极速采集」补充 URL 提取的明确行为：

#### 场景：文本分享与极速采集（更新）

- **当** 用户在第三方应用中选中文本并点击分享到 GleanRead 客户端时
- **那么** 系统通过 `UrlExtractor` 策略链自动尝试提取来源 URL
- **那么** URL 提取成功时展示只读域名徽章；失败时展示可折叠的 URL 输入框
- **那么** 弹出不遮挡全文阅读的轻量浮窗，允许用户输入"想法"与多选"标签"后一键提交至云端
