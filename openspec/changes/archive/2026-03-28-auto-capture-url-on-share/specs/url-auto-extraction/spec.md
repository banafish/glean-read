# 规范：url-auto-extraction

## 目的

在 Android 端提供一个纯本地、无网络的 URL 提取工具，能够从系统分享 Intent 中可靠地
提取被分享文章的原始 URL，支持浏览器「整页分享」和「选中文字分享」两种场景，
以及微信公众号内置浏览器的分享行为。

## 需求

### 需求：策略链提取

系统必须通过一个优先级策略链，按序从 Intent 中提取 URL，首个命中的策略优先返回。

#### 场景：EXTRA_TEXT 整体为 URL
- **当** `Intent.EXTRA_TEXT` 整体以 `http://` 或 `https://` 开头（即为合法 URL）时
- **那么** 直接以该文本作为提取结果返回

#### 场景：EXTRA_SUBJECT 整体为 URL
- **当** `Intent.EXTRA_SUBJECT` 整体以 `http://` 或 `https://` 开头（即为合法 URL）时
- **那么** 直接以该文本作为提取结果返回

#### 场景：EXTRA_STREAM 为 URL
- **当** `Intent.EXTRA_STREAM` 的 Uri 字符串以 `http://` 或 `https://` 开头时
- **那么** 以该 Uri 字符串作为提取结果返回

#### 场景：Intent.clipData 中提取 URL
- **当** `Intent.clipData` 中的条目包含合法 URL 的 Uri 或文本时
- **那么** 提取该 URL 并返回

#### 场景：文本末尾含 URL（选中文字分享）
- **当** `EXTRA_TEXT` 包含文字内容，且通过正则在**文本末尾**匹配到合法 URL 时
- **那么** 以该末尾 URL 作为提取结果返回
- **注意** 末尾判断：提取的 URL 必须是经 trim 后文本的结尾，排除仅出现在正文中间的 URL

#### 场景：文本任意位置含 URL
- **当** `EXTRA_TEXT` 中通过正则在任意位置找到合法 URL，且上述策略均未命中时
- **那么** 以正则匹配到的第一个合法 URL 作为提取结果返回

#### 场景：无法提取
- **当** 所有提取手段均未找到合法 URL 时
- **那么** 返回 `null`，调用方负责显示降级 UI

### 需求：URL 合法性校验

所有被返回的 URL 必须通过合法性校验。

#### 场景：合法 URL 定义

- **当** 字符串以 `http://` 或 `https://` 开头时
- **那么** 视为合法 URL（其余字符串一律不视为合法 URL）

### 需求：无网络调用

URL 提取过程必须完全在本地完成，不得发起任何网络请求。

#### 场景：纯本地执行

- **当** `UrlExtractor.extract(intent)` 被调用时
- **那么** 整个执行过程不得持有任何网络权限或发起任何 HTTP/HTTPS 请求

### 需求：单元可测试性

提取逻辑必须可以通过标准 JUnit 单元测试验证，无需运行在真实设备上。

#### 场景：测试友好接口

- **当** 以 `object UrlExtractor` 形式提供时
- **那么** 其 `extract(intent: Intent)` 函数的所有分支必须可通过 mock Intent 进行单元测试
