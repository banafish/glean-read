# 规范：url-display-in-capture-ui

## 目的

在极速摘录弹窗（`CaptureDialogV2`）中展示 URL 相关的 UI 区域，根据 URL 自动提取的
结果提供两种交互模式：成功时展示只读来源徽章，失败时展示可折叠的编辑输入框。

## 需求

### 需求：来源域名徽章（URL 提取成功）

当系统成功自动提取到 URL 时，必须在弹窗中以只读徽章形式展示来源域名。

#### 场景：展示来源徽章

- **当** `initialUrl` 不为空字符串时
- **那么** 在书摘卡片（`RichExcerptCard`）下方展示 `SourceBadge` 组件
- **那么** `SourceBadge` 显示 🔗 图标及 URL 的 `host` 部分（如 `mp.weixin.qq.com`），不显示完整 URL
- **那么** 徽章为只读，不可点击编辑

#### 场景：徽章样式

- **当** `SourceBadge` 渲染时
- **那么** 以胶囊（RoundedCornerShape）形式呈现，背景为半透明白色，字号不大于 12sp
- **那么** 视觉风格与现有 `TagPill` 协调一致

### 需求：可折叠 URL 输入框（URL 提取失败）

当系统未能自动提取到 URL 时，必须提供折叠式的可选 URL 输入区域。

#### 场景：默认折叠状态

- **当** `initialUrl` 为空字符串时
- **那么** 显示 `CollapsibleUrlInput` 组件，初始状态为**折叠**（仅展示「添加来源链接 +」触发按钮）
- **那么** 折叠状态下保存按钮正常可用，用户可以不填 URL 直接提交

#### 场景：展开并输入

- **当** 用户点击「添加来源链接 +」触发按钮时
- **那么** 输入区域展开，显示 `OutlinedTextField`，用户可粘贴原文 URL
- **那么** 展开的 URL 字段随保存请求一并提交至后端

#### 场景：空 URL 提交

- **当** 用户不展开或展开后输入框为空，点击「保存并继续心流」时
- **那么** 以空字符串作为 URL 字段值正常提交，后端已支持 URL 为空的摘录

### 需求：URL 字段数据流向

URL 最终值（自动提取或用户输入）必须正确传递至保存逻辑。

#### 场景：URL 传递至保存函数

- **当** 用户点击保存时
- **那么** `saveToGleanRead` 调用中的 `url` 参数必须为以下之一：
  - 自动提取到的 URL（`initialUrl`）
  - 用户在 `CollapsibleUrlInput` 中手动填写的 URL
  - 空字符串（用户未填写）
