# 提案：修复摘录界面弹出层阴影与标签展示

## 目的 (Why)
修复 `FastCaptureActivity` 中标签与链接弹出层在显示时的阴影畸变问题（先方后圆），并优化标签较多时的交互体验。

## 目标 (Success)
- [ ] 弹出层显示/隐藏动画期间，阴影边缘始终保持圆角，无视觉畸变。
- [ ] 标签弹出层支持滚动，最大高度能容纳多行标签，不会占用过多屏幕空间。
- [ ] 标签弹出层默认大小能美观地展示至少两行标签。

## 背景 (Context)
当前的弹出层使用 `graphicsLayer` 手动设置阴影，可能是由于 Compose 在动画期间的重新渲染导致形状裁剪未及时应用。同时，`FlowRow` 缺乏容器限制，导致标签过多时排版失控。

## 影响
- `FastCaptureActivity.kt`: 弹出层 UI 组件的相关逻辑。
- 交互流：提升用户在打标时的视觉丝滑感和可用性。
