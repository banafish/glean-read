# 增量规范：Capture V2 现代化 UI

## 目的 (Purpose)
定义 GleanRead 极速摘录组件在视觉表现力与交互层面上的高级要求，确保其通过极致的 UI 审美（「毛玻璃」质感）达成「心流无阻」的摘录体验。

## 需求 (Requirements)

### 需求: 沉浸式毛玻璃视觉 (Immersive Glassmorphism Context)
应用在分享唤起时，其背景必须采用具有通透感且带模糊效果（Blur Effect）的材质，使摘录界面表现为源应用上方的「悬浮层」，极大减少用户的环境切换感。

#### 场景: 分享唤起时的背景融合
- **当** 用户从任何第三方 App 分享文本到 GleanRead 时
- **那么** 弹窗底部必须呈现为圆角（至少 `28.dp`）且带背景模糊的毛玻璃效果，清晰度必须能在浅色与深色背景下均表现优雅。

### 需求: 现代化的书摘卡片化 (Modernized Excerpt Card)
摘录的原文内容必须以独立的「卡片」形式呈现在弹窗核心位置，而非简单的输入框。

#### 场景: 原文内容展示
- **当** 文本被成功提取入弹窗时
- **那么** 它必须以轻度阴影（Tonal Elevation）和优雅排版显示在核心卡片中，卡片应具备书籍、纸张般的质感。

### 需求: 胶囊标签与交互反馈 (Pill Tags & Interaction Feedback)
标签输入应采用「胶囊」化的视觉表现，并在交互时提供丰富的反馈。

#### 场景: 快速打标
- **当** 用户点击预设标签或确认标签时
- **那么** 标签必须以带有呼吸感的动效状态出现在输入框附近，且点击保存按钮时应有微弱的视觉缩放或色彩流动反馈。
