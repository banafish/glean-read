# 规范：标签云与碎片收件箱 (tag-cloud-inbox)

## 概述
管理来自移动端且尚未被编织进“知识树”的所有知识碎片，并通过最近的引用频率生成标签热力云。

## 功能要求
1. **流转机制**: 所有通过捕获接口传入的碎片，初始状态自动归类至 Inbox 收件箱。
2. **标签频次追踪**: 每个被提交的新碎片附带的标签，触发 Redis 或 DB 计数叠加统计近期“热点”。
3. **多维展示**: 
   - 列表视图：按照最新时间排布未经组织的原始碎片。
   - 标签云视图：字体大小与热度成正比交互视图，充当过滤控制条件。
4. **状态移交**: 当碎片被合并到知识树分支之后，需触发从 Inbox 剥离/隐藏的归档操作。

## API 与交互协定
- `GET /api/v1/inbox/fragments`: 拉取待处理碎片分页列表（可带 tag 参数过滤）。
- `GET /api/v1/inbox/tag-cloud`: 拉取带权重的标签热度聚合数据列表，用于渲染前台 UI。
